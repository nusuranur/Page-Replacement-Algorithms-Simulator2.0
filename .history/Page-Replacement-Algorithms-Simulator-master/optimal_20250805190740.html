    OPTIMAL

    Optimal Page Replacement algorithm gives the least number of page faults. It replaces the page that won't be used for the longest time in the future.

    Advantages

    1. Minimum page faults possible.
    2. Efficient benchmark for comparison.

    Disadvantages

    1. Needs future knowledge of reference string.
    2. Not practical for real-time systems.

    ALGORITHM

    1. For each page, check if it's in memory.
    2. If not, find the page in memory that won't be used for the longest time in future and replace it.
