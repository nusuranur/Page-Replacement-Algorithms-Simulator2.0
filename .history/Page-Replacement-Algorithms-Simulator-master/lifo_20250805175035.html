    LAST IN FIRST OUT (LIFO)

    Overview

    Stands for "Last In, First Out." LIFO is an algorithm where the last page entered is the first to be removed. It uses a stack to track pages, with the newest page at the top. When replacement is needed, the top page is removed.

    Advantages
    • LIFO simplifies replacing the newest page.
    • Uses simple data structures.
    • Highly efficient for certain workloads.
    Disadvantages
    • Can delay execution of earlier tasks due to LIFO order.
    Algorithm
    1. Create a stack to store pages in memory.
    2. Maintain the stack and remove the top page for replacement.
